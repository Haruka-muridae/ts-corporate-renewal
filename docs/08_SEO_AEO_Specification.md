# Potenitas SEO・AEO仕様書

**Version:** 1.0.0  
**Status:** Approved with TODOs  
**Last Updated:** 2026-07-22

## 1. 目的

Potenitasの内容を、人、検索エンジン、回答エンジンが同じ事実関係として理解できる状態を作ります。SEOとAEOを別々の小技として扱わず、検索可能な本文、明確なエンティティ、技術的な取得可能性、信頼できる一次情報を共通基盤とします。

## 2. 非保証

- 検索順位、AI回答への引用、AI Overviewへの掲載は保証しない
- 構造化データは表示や掲載を保証しない
- `robots.txt`で許可してもクロールや利用を保証しない
- `llms.txt`を置いてもSEO・AEO効果を保証しない

## 3. サービスの事実定義

- 名称: Potenitas
- 運営: TSアセットマネジメント合同会社の教育事業
- 担当: 齋藤 悠貴
- 区分: 個別指導・伴走型のAI活用支援
- 目的: 特定ツールの正解を暗記させるのではなく、自分で検証・判断し、実際の業務改善を仕組みにし、変化を測定できる状態を支援する
- 提供形態: 1対1、オンライン、60分×月2回、平日チャットは原則24時間以内を目安に返信、月単位

`AIスクール`、`AI講座`という分類は使用しません。資産運用サービスと誤認させないため、運営法人名を単独で提示せず、必ず「教育事業」「個別指導・伴走型のAI活用支援」と隣接させます。

## 4. 基本原則

1. 人に見える本文を最優先する
2. 確定コピーを検索語のために改変しない
3. 事実、推計、仮説を区別する
4. 重要情報をアコーディオンだけに隠さない
5. エンティティ名、価格、提供条件を全ページで一致させる
6. 独自の経験・測定条件・限界を併記する
7. AI向けだけの隠しテキストを作らない
8. 読者の判断を不安や希少性で操作しない

GoogleはAI OverviewsやAI Mode向けに追加の技術要件や特別な最適化は不要と案内しています。通常の検索でインデックス可能であり、スニペット表示が許可され、ユーザーに有用な内容であることを優先します。

## 5. 検索意図

| 区分 | 例 | 対応先 |
|---|---|---|
| 指名 | Potenitas、Potenitas AI、運営会社 | LP、運営者情報 |
| 定義 | AI個別指導とは、伴走型AI活用支援とは | サービス詳細、FAQ、将来記事 |
| 問題解決 | 生成AIを業務で使いたい、AIの回答を検証したい | LP、進め方、記事 |
| 比較 | AI研修と個別指導の違い、AIコンサルとの違い | 比較記事、FAQ |
| 相談 | AI活用を相談したい、個人事業主のAI導入 | LP、問い合わせ |
| 条件確認 | 料金、契約期間、解約、返金、セキュリティ | 料金、FAQ、法務ページ |

検索ボリュームや順位を確認していない語へ数値を付けません。公開後のSearch Consoleデータで、実際のクエリと言葉のずれを検証します。

## 6. エンティティ設計

```text
TSアセットマネジメント合同会社（運営法人）
  └─ 教育事業として運営 → Potenitas（サービス）
       ├─ 担当 → 齋藤 悠貴
       ├─ 提供形態 → 1対1・オンライン・月2回
       ├─ 対象課題 → AI活用、回答検証、業務改善、測定
       └─ 関連概念 → 生成AI、セキュリティ確認、学習記録
```

資産運用との誤認防止策:

- ページtitleと主要説明文に教育サービスであることを含める
- 組織説明とサービス説明を分ける
- `Service`の`provider`で法人との関係を示す
- 金融商品、投資助言、利益保証を連想させる説明を追加しない

## 7. ページ構成の方針

公開初期に必要:

- LP `/`
- 運営者情報 `/about/` または法人サイトへの明確なリンク
- よくある質問 `/faq/`
- 特定商取引法に基づく表記（URLは要確定）
- プライバシーポリシー（URLは要確定）
- 問い合わせ完了ページ `/thanks/`（`noindex`）

後から追加:

- サービス詳細
- 実績・測定方法
- 記事一覧と個別記事
- 編集・訂正・AI利用方針

## 8. コンテンツクラスター

公開後に実際の問い合わせと検索クエリを見て追加します。優先候補:

1. AI個別指導とは何か
2. 生成AI研修・個別指導・コンサルティングの違い
3. 個人事業主がAI導入前に確認すること
4. AIの回答を業務で検証する手順
5. 業務改善の前後をどう測るか
6. 機密情報を生成AIへ入力する前の確認事項
7. 特定ツールに依存しない学び方

記事は、直接回答、条件、具体例、限界、更新日、出典の順で構成します。競合記事の言い換えだけで量産しません。

## 9. 信頼性情報

- 運営法人と問い合わせ先を確認可能にする
- 担当者プロフィールは本人が承認した公開範囲のみ掲載する
- 実績は期間、対象数、測定方法、AI利用の有無、非保証を同じ表示領域で示す
- 更新日と訂正方法を記事に表示する
- 守秘義務で開示できない範囲を明示する
- 法務文書、料金、契約条件を相互に一致させる

E-E-A-Tを単一のランキング要因とは断定しません。利用者と検索システムが情報源を評価するための信頼材料として整えます。

## 10. FAQ方針

FAQは検索表示のために水増しせず、実際の相談前の疑問に回答するために作ります。公開初期候補:

- AI初心者でも相談できますか
- ChatGPT以外の生成AIも対象ですか
- どのような業務を題材にできますか
- 実装やデバッグの代行は含まれますか
- 料金と契約期間を教えてください
- 解約、日割り、返金はどうなりますか
- 未消化の面談はどうなりますか
- 卒業後の支援はありますか
- 機密情報や個人情報はどう扱いますか（回答は方針確定後に公開）

回答は最初の1〜2文で直接答え、続けて条件・例外・参照ページを示します。未確定事項は公開回答にしません。

## 11. 禁止事項

- 隠しテキスト、白文字、画面外配置によるキーワード追加
- 同義語の不自然な列挙
- ほぼ同じ地域・対象別ページの量産
- AI生成文の未確認公開
- 架空のレビュー、顧客数、ロゴ、受賞歴、資格
- 実態と一致しないSchema.org
- 検索エンジンと利用者に異なる本文を返すこと
- AI検索への掲載を保証する表現

## 12. 運用

- 月次: Search Console、Bing、自然検索、問い合わせ内容を確認
- 四半期: title、description、FAQ、内部リンク、古い記載を点検
- 半年: 公式クローラー文書、構造化データ、Next.js、Cloudflare仕様を再確認
- 仕様変更時: 表示本文、メタデータ、JSON-LD、法務文書、計測名を同時更新
