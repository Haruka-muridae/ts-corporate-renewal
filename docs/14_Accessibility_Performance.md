# Potenitas アクセシビリティ・パフォーマンス仕様

**Version:** 1.0.0  
**Status:** Approved  
**Last Updated:** 2026-07-22

## 1. 目標

- WCAG 2.2 Level AAを基準とする
- Core Web Vitalsの「良好」を実ユーザーの75パーセンタイルで目指す
- JavaScriptが失敗しても主要本文とリンクを読める
- Lighthouseの点数だけで合否を判断しない

## 2. セマンティクス

- ページに`h1`は1つ
- 見出し階層を飛ばさない
- ナビ、main、section、footerを適切に使う
- リンクは遷移、ボタンは操作に使う
- リスト、料金、手順を意味に合うHTMLで表現
- 装飾SVGは`aria-hidden="true"`、意味のある図はtitle/descまたは本文説明を付ける

## 3. キーボードとフォーカス

- すべての操作をキーボードで実行可能
- `:focus-visible`を消さない
- 固定ヘッダーや追従CTAでフォーカスを隠さない
- モーダルを採用する場合のみフォーカストラップを実装
- アコーディオンはEnter/Spaceで開閉
- Esc動作は標準的に閉じられるUIだけに適用

## 4. コントラストと文字

- 通常テキスト4.5:1以上
- 大きなテキスト3:1以上
- UI部品・フォーカス表示3:1以上を目標
- 本文16px未満にしない
- 200%拡大で内容や操作を失わない
- 色だけでエラー、状態、料金区分を表さない

## 5. タップ領域

WCAG 2.2 AAの最小ターゲットを意識し、原則44×44 CSS px以上を設計目標とします。例外がある場合も、隣接ターゲットとの間隔を確保します。

## 6. フォーム

- すべての入力に可視ラベル
- 必須・任意をテキストで示す
- `autocomplete`を適切に設定
- エラーを該当項目と概要の両方で提示
- `aria-describedby`で補足・エラーを関連付け
- 送信失敗後に入力内容を不用意に消さない
- 成功・失敗をライブリージョンまたはフォーカス移動で通知

## 7. 動き

- `prefers-reduced-motion: reduce`で不要なアニメーションを停止
- 自動再生、点滅、パララックス、カウントアップを使用しない
- アコーディオンやhoverは短時間で、情報取得を遅らせない

## 8. Core Web Vitals

公式の「良好」基準:

| 指標 | 基準 |
|---|---:|
| LCP | 2.5秒以下 |
| INP | 200ms以下 |
| CLS | 0.1以下 |

実ユーザーの75パーセンタイルで評価します。Lighthouseはラボ検査でありINPを直接測れないため、TBTを参考にしつつ公開後のフィールドデータで確認します。

## 9. プロジェクト内目標

- 初期JSを最小化し、LP本文はServer Componentを基本にする
- 画像へwidth/heightを付けCLSを防ぐ
- OGP画像を本文LCPへ流用しない
- ヒーローに大容量動画・スライダーを置かない
- 第三者スクリプトはGA4等の必要最小限
- フォントウェイトとサブセットを削減
- 主要CSSを阻害しない構成にする

Lighthouse目標はPerformance 90以上、Accessibility 95以上を公開前の目安としますが、100点を必須にして有用性を損ないません。

## 10. 画像

- 実在・実測の情報を補強する画像だけを使用
- 写真はAVIF/WebPを優先し、PNG/JPEGのフォールバック要否を検討
- 装飾画像は空alt
- グラフは同等の数値・注記をHTMLで提供
- ファイル名は内容を示す英小文字とハイフン
- 遅延読み込みはファーストビュー外のみ

## 11. GA4と性能

- GA4は同意・プライバシー方針に従って読み込む
- 可能なら`next/script`の`afterInteractive`等で主表示を妨げない
- タグマネージャーを初期導入しない
- 不要な広告・ヒートマップを入れない
- 導入前後のLCP/TBT/転送量を比較

## 12. セキュリティとプライバシー

- HTTPSのみ
- セキュリティヘッダーをCloudflare Pagesで設定
- GAS側でも入力検証
- 個人情報をGA4イベントへ送らない
- URLやdataLayerへ氏名、メール、自由記述を含めない
- 法的な同意要件は専門家確認事項とする

## 13. 検証幅

最低限: 320、375、390、430、768、1024、1440px。200%拡大、キーボードのみ、スクリーンリーダーの基本読み上げ、reduced motion、低速回線でも確認します。

## 14. 公式参考情報

- Core Web Vitals: https://web.dev/articles/vitals
- CWV thresholds: https://web.dev/articles/defining-core-web-vitals-thresholds
